Devoir MongoDB : Application Pratique
Weiss Manuel

1. Création de la base de données et des collections

D'abord je me connecte à mongosh et je crée/sélectionne la base :

use bibliothequeEnLigne

Puis je crée les 3 collections dont j'ai besoin pour l'appli (une bibliothèque en ligne) :

db.createCollection("auteurs")
db.createCollection("livres")
db.createCollection("avis")

Modélisation

J'ai hésité au début entre tout mettre dans une seule collection "livres" avec l'auteur et les avis imbriqués dedans, mais je me suis dit que ça posait problème si un auteur a plusieurs livres (je duplique ses infos à chaque fois) et si un livre reçoit plein d'avis au fil du temps (le document devient énorme et il faudrait le mettre à jour tout le temps). Du coup j'ai séparé en 3 collections avec des références :

- livres.auteurs = tableau d'ObjectId qui pointent vers auteurs (un livre peut avoir plusieurs auteurs, et un auteur plusieurs livres)
- avis.livreId = un seul ObjectId qui pointe vers le livre concerné

Ça évite de dupliquer les infos et si je dois corriger le nom d'un auteur je le fais à un seul endroit.

2. Insertion de documents

J'ai inséré un auteur, un livre lié à cet auteur, et un avis lié à ce livre :

var auteur = db.auteurs.insertOne({
nom: "Victor Hugo",
dateNaissance: new Date("1802-02-26"),
nationalite: "Francaise"
})

var livre = db.livres.insertOne({
titre: "Les Miserables",
anneePublication: 1862,
genres: ["Roman", "Historique"],
auteurs: [auteur.insertedId]
})

db.avis.insertOne({
evaluation: 5,
commentaire: "Un chef-d'oeuvre intemporel.",
livreId: livre.insertedId
})

J'ai ajouté un deuxième auteur/livre/avis pour avoir un peu plus de données à tester, notamment un livre publié après 2000 et des avis avec des notes différentes :

var auteur2 = db.auteurs.insertOne({
nom: "Michel Bussi",
dateNaissance: new Date("1965-04-01"),
nationalite: "Française"
})

var livre2 = db.livres.insertOne({
titre: "Nymphea Noir",
anneePublication: 2011,
genres: ["Policier", "Thriller"],
auteurs: [auteur2.insertedId]
})

db.avis.insertOne({
evaluation: 2,
commentaire: "Trop prévisible à mon goût.",
livreId: livre2.insertedId
})

db.avis.insertOne({
evaluation: 4,
commentaire: "Très bonne intrigue, rebondissements surprenants.",
livreId: livre2.insertedId
})

3. Requêtes de lecture

Livres publiés après 2000 :

db.livres.find({ anneePublication: { $gt: 2000 } })

Ça me retourne bien "Nymphea Noir" (2011) et pas "Les Miserables" (1862).

Avis avec une note >= 4 :

db.avis.find({ evaluation: { $gte: 4 } })

4. Mise à jour de documents

Je récupère d'abord le livre que je veux modifier grâce à son titre :

var livreCible = db.livres.findOne({ titre: "Les Miserables" })

Puis j'ajoute un genre au tableau existant. Au début j'avais essayé avec $set mais ça remplaçait tout le tableau genres au lieu de juste ajouter un élément, du coup j'ai utilisé $push qui ajoute à la fin sans toucher aux genres déjà présents :

db.livres.updateOne(
{ _id: livreCible._id },
{ $push: { genres: "Classique" } }
)

5. Suppression de documents

Je supprime l'avis le moins positif en le ciblant par son commentaire exact :

db.avis.deleteOne({ commentaire: "Trop prévisible à mon goût." })

6. Indexation

Je crée un index sur anneePublication puisque c'est un champ sur lequel je fais des recherches (partie 3), ça évite à MongoDB de parcourir toute la collection à chaque fois :

db.livres.createIndex({ anneePublication: 1 })

Je vérifie que l'index soit bien là :

db.livres.getIndexes()

7. Agrégation

Pour compter le nombre de livres par genre, j'utilise un pipeline en 3 étapes. Le plus dur à comprendre pour moi ça a été $unwind : en gros si un livre a 2 genres, $unwind va dupliquer temporairement ce livre en 2 documents (un par genre) pour pouvoir les compter séparément ensuite.

db.livres.aggregate([
{ $unwind: "$genres" },
{ $group: { _id: "$genres", nombreLivres: { $sum: 1 } } },
{ $sort: { nombreLivres: -1 } }
])

Résultat obtenu après avoir ajouté le genre Classique à Les Miserables :

- Roman : 1
- Historique : 1
- Classique : 1
- Policier : 1
- Thriller : 1

Toutes les commandes ont été testées dans mongosh sur la base bibliothequeEnLigne, ça fonctionne.
